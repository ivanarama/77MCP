# mcp-1c77

MCP-сервер для анализа метаданных конфигураций 1С:Предприятие 7.7.

Позволяет LLM-клиентам (например, Claude Code) получать доступ к структуре конфигурации 1С 7.7, включая справочники, документы, регистры, модули форм и исходный код модулей объектов.

## Особенности

- **Парсинг файлов 1Cv7.MD** — чтение метаданных напрямую из файлов конфигурации
- **Web-интерфейс** — удобная загрузка файлов через браузер
- **MCP over SSE** — интеграция с Claude Code и другими MCP-клиентами
- **Поддерживаемые объекты:**
  - Справочники (Catalogs)
  - Документы (Documents)
  - Регистры (Registers)
  - Перечисления (Enums)
  - Константы (Constants)
  - Отчёты/Обработки (Reports)
  - Журналы документов (Document Journals)
  - Виды расчётов (CalcVars)

```bash
git clone <repo-url>

```

## Использование

### Запуск сервера

```bash
python -m mcp_1c77
```

Сервер запустится на `http://localhost:8080`.

### Web-интерфейс

1. Откройте `http://localhost:8080` в браузере
2. Перетащите файл `1Cv7.MD` в зону загрузки
3. После загрузки вы увидите статистику по конфигурации

### Интеграция с Claude Code

Добавьте MCP-сервер в Claude Code:

```bash
claude mcp add --transport sse 1c77-metadata http://localhost:8080/sse
```

Теперь вы можете использовать инструменты MCP в диалоге с Claude.

## MCP инструменты

| Инструмент | Описание |
|------------|----------|
| `reload_configuration` | Перезагрузить конфигурацию или загрузить другой файл |
| `list_objects` | Список объектов метаданных с возможностью фильтрации по типу |
| `get_object` | Детальная информация об объекте (реквизиты, табличные части) |
| `get_module` | Получить исходный код модуля объекта |
| `get_form` | Получить описание формы объекта |
| `search` | Поиск по объектам (имя, синоним, комментарий) |
| `get_configuration_info` | Общая информация о конфигурации |
| `validate_field_path` | Проверить валидность пути обращения к реквизиту (напр. "Товар.Артикул") |
| `validate_query` | Проверить все пути обращений к реквизитам в тексте запроса/кода |
| `search_field` | Найти все объекты, содержащие реквизит с данным именем |
| `get_objects_batch` | Пакетное получение метаданных нескольких объектов за один вызов |

### Примеры использования

В Claude Code после подключения MCP-сервера:

```
# Получить список всех документов
list_objects(object_type="Документ")

# Получить детальную информацию о документе
get_object(object_type="Документ", name="РасходнаяНакладная")

# Проверить путь обращения к реквизиту
validate_field_path(object_type="Документ", name="РасходнаяНакладная", path="Товар.Артикул")

# Найти все объекты с реквизитом "Сумма"
search_field(field_name="Сумма")

# Проверить запрос на ошибки в путях реквизитов
validate_query(query_text="SELECT Документ.Товар.Артикул FROM Документ.РасходнаяНакладная AS Документ")
```

## Формат 1Cv7.MD

Файл конфигурации `1Cv7.MD` 1С:Предприятие 7.7 представляет собой:
- **OLE-контейнер** — Compound Document Binary Format
- **Внутренняя структура** — текстовое представление в скобочном синтаксисе
- **Кодировка** — обычно Windows-1251

Парсер выполняет:
1. Извлечение содержимого из OLE-контейнера
2. Разбор скобочного синтаксиса в дерево объектов
3. Парсинг модулей форм и объектов
4. Построение связей между объектами
